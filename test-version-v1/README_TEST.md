# 🧪 测试环境说明 - 正式版1.0副本

## 版本信息
- **版本**: 正式版1.0 (测试副本)
- **环境**: 测试环境，与生产环境完全隔离
- **目的**: 在不影响生产环境的情况下测试和调试
- **状态**: 与正式版1.0功能完全一致

## 🎯 测试环境特点

### 1. 与正式版完全一致
- ✅ **相同代码** - 正式版1.0的完整副本
- ✅ **相同功能** - 所有学习功能保持不变
- ✅ **相同界面** - 用户体验完全一致
- ✅ **相同数据** - 使用相同的单词和短语数据

### 2. 最小化测试干扰
- 🧪 **测试标识** - 右上角显示测试环境标识
- 📝 **环境提示** - 顶部显示测试环境说明
- 🔧 **调试工具** - 双击页面显示调试信息
- 📱 **设备检测** - 自动记录设备信息

### 3. 完全独立的环境
- 🛡️ **数据隔离** - 测试环境数据独立存储
- 🔒 **安全隔离** - 不会影响生产环境
- 🚀 **部署独立** - 可以独立更新和测试
- 📊 **监控独立** - 独立的错误日志和性能监控

## 📱 测试功能

### 核心学习功能
1. **每日词汇** - 单词学习和记忆
2. **常用短语** - 实用对话练习
3. **发音练习** - Azure TTS发音支持
4. **威尔士文化** - 文化背景学习

1. **发音按钮** - 点击播放单词发音
2. **单词切换** - 上一个/下一个单词
3. **闪卡功能** - 加入闪卡和标记已学
4. **导航切换** - 侧边栏模块切换

## 🔧 调试功能

### 1. 调试面板
- **显示方式**: 双击页面任意位置
- **内容**: JavaScript错误、设备信息、测试结果
- **位置**: 右下角浮动面板

### 2. 设备检测
- 自动检测iOS/iPhone设备
- 记录屏幕尺寸和触摸支持
- 显示用户代理信息

### 3. 错误监控
- 捕获JavaScript运行时错误
- 记录到调试面板
- 不影响正常功能使用

## 🚀 测试流程

### 第一步：基础功能测试
1. 访问测试环境地址
2. 等待页面完全加载
3. 检查所有界面元素显示正常
4. 测试发音按钮功能
5. 测试单词切换功能

### 第二步：交互测试
1. 点击侧边栏不同模块
2. 测试所有按钮的点击反馈
3. 检查触摸事件支持
4. 验证数据加载和显示

### 第三步：问题诊断
1. 双击页面打开调试面板
2. 查看设备信息和错误日志
3. 记录发现的问题
4. 对比正式版的表现

## ⚠️ 注意事项

### 1. 测试环境用途
- 功能验证和回归测试
- 兼容性测试（特别是iPhone）
- 性能测试和优化
- 新功能预发布测试

### 2. 数据安全
- 测试环境使用独立数据存储
- 不会影响生产环境数据
- 测试数据可能被重置
- 建议不要使用重要数据

### 3. 稳定性
- 测试环境可能不稳定
- 可能包含实验性功能
- 更新频率较高
- 仅供测试使用

## 🔄 更新策略

### 1. 同步策略
- 测试环境从正式版同步
- 保持功能一致性
- 独立进行问题修复
- 验证后同步到正式版

### 2. 测试策略
- 先在测试环境验证修复
- 确认无副作用后部署到正式版
- 保持两个环境的功能同步
- 定期进行回归测试

## 📞 问题反馈

### 1. 测试环境问题
- 记录具体操作步骤
- 截图显示问题
- 查看调试面板错误信息

- 在测试环境复现问题
- 验证修复方案
- 确认无副作用后更新正式版
